# Day 5: Network Traffic Analysis with Wireshark 👀

## Introduction
On Day 5 of my cybersecurity journey, I used Wireshark to capture and analyze network traffic. This helped me understand how data moves across a network in real time.

---

## What is Wireshark?
Wireshark is a network protocol analyzer that allows users to capture and inspect data packets traveling across a network.

---

## What I Did
- Opened Wireshark
- Selected an active network interface
- Started capturing packets
- Visited a website to generate traffic
- Applied filters to analyze specific protocols

---

## Filters I Used

- `dns` → To view domain name requests
- `http` → To view web traffic

---

## What I Observed
- DNS requests when accessing websites
- Communication between my system and external servers
- Different protocols working together

---

## What I Learned
- How to capture live network traffic
- How to filter and analyze packets
- The importance of monitoring network activity in cybersecurity

---

## Challenges I Faced
I am still learning to extract and understand information from packets

---

## Reflection
Seeing live network traffic was very interesting. It helped me understand what happens behind the scenes when I browse the internet.

---

## Conclusion
Wireshark is a powerful tool for analyzing network activity, and it plays a key role in cybersecurity investigations.
